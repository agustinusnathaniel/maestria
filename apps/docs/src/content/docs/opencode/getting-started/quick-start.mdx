# Quick Start

Once you've installed the plugin, all agents and rules are loaded automatically in every session — no extra commands, no manual setup.

## Start a Session

Launch OpenCode as usual. The plugin wires itself in. Just start working.

## Delegate to a Specialist

Call any of the 7 specialists directly:

```markdown
@adventurer Explore the auth module and trace the login flow.
```

For multi-step work, hand it to the orchestrator:

```markdown
@orchestrator I need to add rate limiting to the API. Plan and implement it.
```

The orchestrator breaks the task down, delegates to the right specialists, and integrates the results.

## Pipeline Flow

A typical multi-step task moves through the agent system like this:

```
@orchestrator
  ├─ Thinker (recon/design/planning/analysis)
  │   ├─ @adventurer → Explore codebase
  │   ├─ @architect  → Design approach
  │   ├─ @planner    → Create implementation plan
  │   └─ @diagnose   → Analyze bugs
  ├─ Worker (implementation/documentation)
  │   ├─ @builder → Implement
  │   └─ @writer  → Document
  └─ Verifier (validation)
      └─ @reviewer → Review changes
```

Each agent hands off to the next. The orchestrator keeps everything moving.

## Workflow Modes

The orchestrator supports three modes that change how it delegates work:

- **`fein`** — Default role-based pipeline with acceptance gates. Best for production changes.
- **`sonar`** — Research only, no implementation. Best for understanding code.
- **`blitz`** — Fast implementation, skip gates. Best for quick fixes.

Type the keyword anywhere in your message:

```
fein add rate limiting to the API
```

See [Workflow Patterns](/opencode/guide/workflow-patterns/) for detailed examples.
