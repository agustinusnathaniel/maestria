# Changelog

## 0.3.5

### Patch Changes

- **The orchestrator is now a strict dispatcher.** Its read-side tool permissions (`read`, `glob`, `grep`, `lsp`, `webfetch`) are removed, and the opening prompt is rewritten as a dispatcher mandate: `task()` and `question()` are the only tools for making progress. The 7 specialist subagents keep full read-side access for the work they pick up.
- **Three new directives added to global rules.** "Webfetch may hang — don't block on it", "CLI references — use local tools first", and "Local files — read directly" are added, closing 21 directive-coverage gaps (3 × 7 specialists) so the guidance now applies uniformly.

## 0.3.4

### Patch Changes

- Audit and update skill prescriptions across all 7 agents. Fix renames (`diagnose` → `diagnosing-bugs`, `write-a-skill` → `writing-great-skills`), remove the dead `zoom-out` skill, correct 10 wrong source repos (e.g., ADRs → wshobson/agents, review-logging-patterns → hugorcd/evlog), drop skills from stale sources (sickn33/antigravity-awesome-skills, refoundai/lenny-skills), and add 8 new skills from mattpocock/skills and anthropics/skills with cross-references across the appropriate agents.

## 0.3.3

### Patch Changes

- Audit and sort agent skill prescriptions across all 7 agents. Alphabetically sorts "Load on trigger" skill lists and adds missing entries (e.g., `session-handoff` to adventurer, `prioritizing-roadmap` and `technical-roadmaps` to planner).

## 0.3.2

### Patch Changes

- Strengthen orchestrator delegation rules. Rule #1 expanded to explicitly forbid using shell for implementation. New Rule #2: "Shell is not a workaround" — catches the common failure mode of substituting shell commands for delegation.

## 0.3.1

### Patch Changes

- Restrict orchestrator task permissions to 7 registered subagents. Changed from `"*": allow` to deny-by-default with explicit allow-list for only the 7 specialists.

## 0.3.0

## 0.2.0 (2026-06-12)

### Features

- **Initial plugin release** — `@maestria/opencode` with full agent system and rule injection
- **7 specialist subagents** — adventurer, architect, builder, diagnose, planner, reviewer, writer
- **Orchestrator agent** — `@orchestrator` with `mode: all` for managing multi-step workflows
- **Global rules injection** — Cross-cutting rules (orchestration, delegation, context management) injected into every session
- **Config hook** — Reads bundled agent `.md` files, parses frontmatter, registers agents programmatically
- **Session compacting hook** — Preserves task tracking state across compaction events
- **Agent cross-references** — Every agent lists related agents with delegation triggers
- **Skill discovery guidance** — Every agent follows a check → use → suggest pattern for skills
- **Conventional Comments** — Reviewer output uses standard labels (praise, suggestion, issue, nitpick, question)
- **`opensrc` pattern** — Global rule for analyzing reference repos without API calls
