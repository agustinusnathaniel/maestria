## Why Maestria?

Every AI coding tool needs agent configuration - prompts, rules, permissions, and workflows. Maestria encodes effective AI-engineering patterns into installable, self-wiring plugins. No manual setup. No copy-pasting configs. Just install and go.

## Why It Works

Maestria's **Harness Engineering** approach uses a dispatcher that routes work to
7 specialists instead of one session that handles everything. The pipeline
structures work into stages: reconnaissance, design, implementation, and review.

**Primary benefits.** Global rules keep behavior consistent across projects. The
maker/checker split prevents self-approval bias. The mode system (fein, sonar,
blitz) lets you choose quality or speed per turn. Portable plugins work across
OpenCode, Kimi Code, and Pi.

**Emergent benefits.** Context engineering allocates a fresh session per
specialist instead of sharing one window. Parallel fan-out shrinks wall-clock
time. One-shot work catches mistakes before they compound. Bounded failure
replaces infinite loops with structured escalation.

## The Ecosystem

Maestria provides the same AI engineering methodology across three platforms:

| Plugin                  | Platform        | Best for                                             |
| ----------------------- | --------------- | ---------------------------------------------------- |
| **@maestria/opencode**  | OpenCode SDK    | Full subagent support, mature plugin ecosystem       |
| **@maestria/kimi-code** | Kimi Code       | Declarative setup, swarm fan-out, no build step      |
| **@maestria/pi**        | Pi coding agent | Review mode, session persistence, rich extension API |

**Which should I install?** Choose the plugin that matches your AI coding agent platform. The methodology is identical - each plugin adapts the same agent prompts and workflow rules to its platform's native primitives. If you switch platforms, your workflow patterns carry over.

## How It Works

Maestria plugins are built on a simple architecture:

1. **Config hook** - Reads bundled agent definitions and registers them with the coding agent
2. **Rules injection** - Global directives appended to every session's system prompt
3. **Session hooks** - Preserves task state across compaction events

This means you get consistent behavior from the first prompt - no manual agent configuration required.
