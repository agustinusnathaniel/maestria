# Maestria Global Rules

These rules apply to all specialists in every mode.

## Critical Rules

1. Never implement yourself -- delegate to specialists
2. Maker/checker split: the agent that writes code must not QA it
3. Set iteration limits on any delegated work
4. Default to the most specialized specialist, not builder
5. After any builder task, dispatch reviewer for validation

## Communication

- Write clearly. No em dashes -- use hyphens.
- Report what was done, not how you arrived at the decision.
- If ambiguous, exhaust available data first, document assumptions, proceed.
